================================================================================
       MUSHROOM DISEASE CLASSIFICATION - RESEARCH PLAN & WORKFLOW
================================================================================
Dataset: Mushroom Disease Dataset (Healthy, Single Infected & Mixed Infected)
Location: Mushroom Development Institute, Savar, Dhaka, Bangladesh
Captured: April 2025 | Device: iPhone 11 Pro Max | Total Images: 761
Classes: Healthy | Single Infected (Green/Black Mold) | Mixed Infected
Annotation Status: Not annotated (annotation required before training)

References: Vidanapathirana et al. (2023), Shilpashree et al. (2025),
            Wongpanya et al. (2025), Guragain et al. (2024)
================================================================================


--------------------------------------------------------------------------------
PHASE 1: DATASET PREPARATION & ANNOTATION
--------------------------------------------------------------------------------
Goal: Convert raw, unlabeled images into a clean, structured, labeled dataset.

Steps:
  1.1  Audit & Inventory
       - Verify all 761 images are readable and not corrupted
       - Count images per folder: Healthy / Single_Infected / Mixed_Infected
       - Document any duplicates or near-duplicates and remove them

  1.2  Manual Annotation / Labeling
       - Confirm class labels for each image:
           Class 0 → Healthy
           Class 1 → Single Infected  (suspected Trichoderma / Aspergillus /
                                        Rhizopus — green mold or black mold)
           Class 2 → Mixed Infected   (healthy + mold OR mold + mold in
                                        same image)
       - Tool recommendation: LabelImg or Roboflow (free tier)
       - For classification only: folder-based labels are sufficient
       - For object detection (YOLO): draw bounding boxes around mold regions
       - Save annotations in YOLO or Pascal VOC format

  1.3  Dataset Split
       - Split per class to preserve class balance:
           Train  : 70%  (~533 images)
           Val    : 15%  (~114 images)
           Test   : 15%  (~114 images)
       - Use stratified split to ensure each class is represented equally
         in all three splits

  NOTE: 761 images is a small dataset. Augmentation (Phase 2) is critical.

Deliverable: Organized folder structure —
  dataset/
    train/  healthy/  single_infected/  mixed_infected/
    val/    healthy/  single_infected/  mixed_infected/
    test/   healthy/  single_infected/  mixed_infected/


--------------------------------------------------------------------------------
PHASE 2: PREPROCESSING & DATA AUGMENTATION
--------------------------------------------------------------------------------
Goal: Standardize inputs and artificially expand the dataset to reduce
      overfitting (inspired by all 4 reference papers).

Steps:
  2.1  Preprocessing
       - Resize all images to a uniform size: 224x224 px (standard for
         VGG16, ResNet50, InceptionV3, DenseNet201)
       - Normalize pixel values to [0, 1] by dividing by 255
       - Convert all images to RGB (3-channel) if not already

  2.2  Augmentation (applied only to training set)
       - Horizontal flip
       - Vertical flip
       - Rotation: up to 30 degrees
       - Zoom: up to 20%
       - Shear transformation: up to 15%
       - Brightness adjustment: +/- 20%
       - Random noise addition (Gaussian)
       - Width/height shift: up to 10%
       - Target augmented training set size: ~2,000–2,500 images

  2.3  Tools
       - Keras ImageDataGenerator  OR  Albumentations library (Python)
       - Apply augmentation on-the-fly during training (saves disk space)

Deliverable: Preprocessing + augmentation pipeline (Python script)


--------------------------------------------------------------------------------
PHASE 3: MODEL SELECTION & TRAINING
--------------------------------------------------------------------------------
Goal: Train and compare multiple deep learning models for classification.
      This is the core experimental phase.

Strategy: Transfer Learning (pre-trained on ImageNet) + Fine-tuning
          (consistent with Papers 1, 3, 4)

  3.1  Baseline CNN (Custom)
       - Build a lightweight custom CNN as a performance baseline
       - 3–4 Conv2D + MaxPooling layers → Flatten → Dense → Softmax (3 classes)
       - Helps understand what pre-trained models add

  3.2  Transfer Learning Models (compare all)
       ┌─────────────────┬────────────────────────────────────────────────┐
       │ Model           │ Why / Reference                                │
       ├─────────────────┼────────────────────────────────────────────────┤
       │ VGG16           │ Paper 1 (Vidanapathirana) — simple, robust     │
       │ ResNet50        │ Papers 1, 3 — strong residual connections       │
       │ InceptionV3     │ Papers 1, 3 — multi-scale feature extraction   │
       │ DenseNet201     │ Papers 3, 4 — best performer (92.5% acc)       │
       │ EfficientNetB0  │ Modern, lightweight, high accuracy (bonus)     │
       └─────────────────┴────────────────────────────────────────────────┘

  3.3  Training Configuration
       - Framework: TensorFlow / Keras (Python)
       - Optimizer: Adam
       - Loss Function: Categorical Cross-Entropy (3-class softmax output)
       - Batch Size: 32
       - Epochs: 50 (with early stopping, patience=10)
       - Learning Rate: 0.0001 (fine-tuning)
       - Fine-tuning strategy:
           Step 1 — Freeze all base model layers, train top classifier
                    (5–10 epochs)
           Step 2 — Unfreeze last 20–30 layers, retrain with low LR
                    (remaining epochs)
       - Callbacks: EarlyStopping, ModelCheckpoint (save best weights),
                    ReduceLROnPlateau

  3.4  Optional: Object Detection Variant
       - If bounding box annotations are completed (Phase 1.2):
         Train YOLOv5 or YOLOv8 for localization of mold regions
         (inspired by Paper 2 — Shilpashree, 94% precision)
       - This enables real-time detection, not just classification

Deliverable: Trained model weights (.h5 / .pt files) for each architecture

--------------------------------------------------------------------------------
KEY CHALLENGES & MITIGATION
--------------------------------------------------------------------------------
  Challenge 1: Small Dataset (761 images)
  → Mitigation: Heavy augmentation + Transfer Learning (pre-trained weights)
               + Consider adding public mushroom disease images if available

  Challenge 2: Class Imbalance (unknown exact split per class)
  → Mitigation: Count images per class first; use class_weight parameter
               in Keras training if imbalanced

  Challenge 3: "Mixed Infected" is ambiguous (multi-label nature)
  → Mitigation: Treat as a third distinct class for simplicity (multi-class);
               OR explore multi-label classification as future work

  Challenge 4: No Annotations
  → Mitigation: Folder-based labels sufficient for image classification;
               only need bounding boxes if doing object detection (YOLO)

  Challenge 5: Suspected mold types (Trichoderma / Aspergillus / Rhizopus)
               not confirmed in metadata
  → Mitigation: Focus on visual contamination detection, not species ID;
               note limitation in paper and recommend lab confirmation

  - Small dataset + strong augmentation pipeline is a reproducible contribution
